The Odin Project

Rock Paper Scissor Game

by karl rickard vestin
3 Months Yes, I've been tinkering with this project for that long. I’ve dipped in and out and even started an elaborate alternative version, which I worked on (and still do sometimes) for about a month. My aim is to learn and bring ideas to life, so whatever I dream up, I’m determined to figure out and implement, like the idea of a firework display if you win the game.
83 Commits I could've done more, but this was great practice for the whole GitHub experience, and I learned a lot!
0 Experience Before starting The Odin Project, I had pretty much no experience in coding. I did some very minor HTML markup way back in the last millennium, but that just meant I knew what <br> does
15+ Branches Was used (roughly) while building this. I made a point of creating new feature branches, which helped immensely with navigating the project’s history. At the beginning, I didn’t know how branches, merging, and rebasing worked, so I kept manual copies of files, which was a total mess. Once I got the hang of branching and especially rebasing, my development environment became much smoother and more manageable.
>1k Lines of script Probably more like 1500, ha! There’s a lot of junk in there that should be cleaned up.
18 Event Listeners These were really fun and handy to learn. I especially liked listening for transitions and animation starts and ends to trigger other elements' actions.
3 Canvas Elements The HTML Canvas element is pure magic and such a versatile tool for creating art and movement on the page. I used two canvases for the "drip effect" displayed when the player loses a game. I used another, much smaller, canvas for the line that "cuts out" the 3D rotating text.
1 Variable Font I was instantly curious about variable fonts once I started to get the hang of JavaScript. The idea of fonts fluidly changing widths and weights was too tempting not to try. Maybe I overdid it, but hey, I’m having fun here!
198 Variables That’s what it says when I search for “let” in my script file. It’s not a scientific study since it also catches words like “Let’s Play,” but considering there are a few consts in there too, it’s probably not too far off.
40 Functions Not counting arrow functions.
60 Console.log's Currently spread around my script file, though there have definitely been more.
3 Grid Layouts The most prominent one is this page.
21 Flex Layouts Definitely my most used display setting. It’s so useful for organizing elements within a certain space.
9 Media Queries At the moment of writing. I know I'm not using these to best effect and I'm probably making things too complicated but hey I need something to dive deeper into for next project.
5 Classes These where used for the firework, and drip effects, as blue prints for creating new objects within said effects.
1.8k Lines of CSS I was really surprised to see it this long to be honest.
228 Lines of HTML This is probably the least wasteful file of them all.
1 SVG Element Or rather several elements but one svg project so to say. The firework display is made by animating svg elements.
All the above is subject to many grains of salt, as things change constantly whilst working on this project
